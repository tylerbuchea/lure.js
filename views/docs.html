Lure.js usage documentation

.init

Here we create a new instance of Lure and use init which binds the initial routes to the onpopstate event. 

	
			var App = new Lure( $("[x-view]") );
			App.init();

.route

Wires together the current URL location, view template, and controller. 


			App.route("/", {

			    templateUrl: "template.html",
			    controller: "TemplateCtrl"

			});

.controller

Breaks the code into functional pieces and can be called from the DOM with the x-ControllerName="value" attribute or from within a route using the controller property. 


			App.controller("name", 
				function(model, el, value, controllers) {

				    $(el).on("click", function() {

				        window.location.href = value;

				    });	

				});

.http

Is just a wrapper for the JQuery $.ajax call that automatically updates the view after the data has been returned. All the same $.ajax arguments apply. 


			App.http({

			    url: 'https://api.github.com/users/tylerbuchea/' +
			         'gists?callback=?',
			    dataType: 'json',
			    success: function(data) {
			        model.myinfo.github = data;
			    }
				
			});
